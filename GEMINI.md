# MailMind Project Context

MailMind is an AI-powered email daemon that transforms your inbox into an interface for AI models. It listens for incoming emails, processes instructions using various AI backends (CLI or API), and automatically replies with the results.

## 🏗️ Architecture & Core Components

- **`email_daemon.py`**: The central engine. It handles IMAP connection (supporting IDLE for real-time push or polling), email parsing, AI invocation, and SMTP replying. It also includes a task scheduler for periodic jobs.
- **`manage.sh`**: The primary entry point for users. It provides an interactive setup wizard, service lifecycle management (start/stop/restart), logs viewing, and systemd integration.
- **`webui/`**: A FastAPI-based web interface (accessible via `bash manage.sh webui`) for monitoring status, configuring settings, and managing tasks visually.
- **`tasks.json`**: Persistence layer for scheduled tasks.
- **`reports/`**: Archive directory for results generated by scheduled tasks (e.g., daily news summaries, system status reports).

## 🛠️ Technical Stack

- **Language**: Python 3.8+
- **Protocols**: IMAP (Receiving), SMTP (Sending).
- **AI Backends**:
    - **CLI**: Claude Code (`claude`), Gemini CLI (`gemini`), Codex, Qwen.
    - **API**: DeepSeek, OpenAI (GPT-4o), Anthropic (Claude 3.5 Sonnet), Gemini API, Qwen API.
- **Key Libraries**: `imapclient` (IDLE mode), `requests` (API calls), `markdown` (HTML rendering), `fastapi` (Web UI).
- **Search Integration**: Supports DuckDuckGo, Google, Bing, and Wikipedia for web-augmented tasks.

## 🚀 Key Workflows

### 1. Setup & Configuration
Run the interactive wizard to configure your mailbox and AI backend:
```bash
bash manage.sh setup
```
This generates a `.env` file with your credentials and preferences.

### 2. Service Management
- **Start**: `bash manage.sh start`
- **Stop**: `bash manage.sh stop`
- **Web UI**: `bash manage.sh webui start` (defaults to http://localhost:8000)
- **Systemd**: `bash manage.sh install` for auto-start on boot.

### 3. Usage
Send an email to the configured address. If your sender address is in the `ALLOWED` whitelist, MailMind will:
1. Parse the subject and body.
2. Detect if it's a direct command or a scheduling request (e.g., "Every day at 8:00...").
3. Execute the AI/Task.
4. Reply with a formatted email (AI will automatically generate a relevant subject).

## 📅 Scheduled Tasks
MailMind supports several task types that can be triggered via email instructions:
- `ai_job`: Custom AI analysis or generation.
- `weather`: Weather reports.
- `news`: News aggregation.
- `web_search`: Periodic web searching.
- `system_status`: OS metrics (CPU, Memory, Disk).
- `report`: Combined daily/weekly reports.

## 🛡️ Security & Conventions
- **Whitelist**: Only emails from addresses in `MAIL_<NAME>_ALLOWED` are processed.
- **Credentials**: Never commit `.env`, `token_*.json`, or `credentials_*.json`.
- **Logging**: Logs are stored in `daemon.log` and `webui.log`.
- **Coding Style**: Follow standard Python PEP 8. Use `log.info/debug/error` within `email_daemon.py` for consistency.
